### 📝 Tóm tắt 5 câu đã làm
1. **Câu 1:** A hospital recently deployed a RESTful API with Amazon API Gateway and AWS Lambda. The hospital uses API Gateway and Lambda to upload reports that are in PDF format and JPEG format. The hospital needs to modify the Lambda code to identify protected health information (PHI) in the reports.
Which solution will meet these requirements with the LEAST operational overhead?

- Kiến thức rút ra:
    Để least ops thì cần serverless tối đa.
    Quét thông tin từ report thì dùng AWS Text Extract để phân tích text từ file.
    Để identify thông tin PHI y tế thì dùng Amazon Comprehend Medical chuyên dụng.

2. **Câu 2:** A company uses AWS Organizations to manage multiple AWS accounts for different departments. The management account has an Amazon S3 bucket that contains project reports. The company wants to limit access to this S3 bucket to only users of accounts within the organization in AWS Organizations.
Which solution meets these requirements with the LEAST amount of operational overhead?
- Kiến thức rút ra:
    Để hạn chế thì cần dùng S3 policy.
    Điều kiện pass policy là dùng aws PrincipalOrgID global condition key vì mỗi Org sẽ có key riêng, đúng key sẽ tự pass policy.
    Do muốn access cho toàn Org giống nhau nên dùng đc, còn nếu chia từng phòng ban phải tạo OU , nhưng tốn effort hơn.

3. **Câu 3:** A company is launching a new application and will display application metrics on an Amazon CloudWatch dashboard. The company's product manager needs to access this dashboard periodically. The product manager does not have an AWS account. A solutions architect must provide access to the product manager by following the principle of least privilege.
Which solution will meet these requirements?
- Kiến thức rút ra:
    Share the dashboard from the CloudWatch console. Enter the product manager's email address, and complete the sharing steps. Provide a shareable link for the dashboard to the product manager.
    Cloudwatch có thể share link vs email để có quyền xem, và phân quyền trên từng dashboard nhất định.
    Đảm bảo ít quyền và k cần login aws.

4. **Câu 4:** A company has an application that ingests incoming messages. Dozens of other applications and microservices then quickly consume these messages. The number of messages varies drastically and sometimes increases suddenly to 100,000 each second. The company wants to decouple the solution and increase scalability.
Which solution meets these requirements?
- Kiến thức rút ra:
    Publish the messages to an Amazon Simple Notification Service (Amazon SNS) topic with multiple Amazon Simple Queue Service (Amazon SOS) subscriptions. Configure the consumer applications to process the messages from the queues.
    Đề cần tách biệt producer và consumer, do đó nên dùng pattern 1 SNS bắn noti đến multiple SQS.
    Sau đó sẽ có các service xử lí mess từ SQS.
    Dùng SNS vì nó nhanh, xử lí nhiều mess, SQS thì auto scale.

5. **Câu 5:** A company runs its infrastructure on AWS and has a registered base of 700,000 users for its document management application. The company intends to create a product that converts large .pdf files to .jpg image files. The .pdf files average 5 MB in size. The company needs to store the original files and the converted files. A solutions architect must design a scalable solution to accommodate demand that will grow rapidly over time.
Which solution meets these requirements MOST cost-effectively?
- Kiến thức rút ra:
    Save the .pdf files to Amazon S3. Configure an S3 PUT event to invoke an AWS Lambda function to convert the files to .jpg format and store them back in Amazon S3.
    Để lưu data thì auto cần S3, để convert file và lưu lại cả 2 loại (original + converted) thì dùng combo S3 -> S3 event -> trigger lambda -> lambda convert and save into S3.
