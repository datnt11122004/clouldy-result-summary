# 1. AWS Shield / AWS Shield Advanced
Chi tiết: Dịch vụ bảo vệ chống lại các cuộc tấn công DDoS ở tầng Mạng (Layer 3) và tầng Giao vận (Layer 4). Bản Standard có sẵn và miễn phí, nhưng bản Advanced cung cấp khả năng phát hiện chủ động, giảm thiểu các cuộc tấn công tinh vi theo thời gian thực và có đội ngũ chuyên gia hỗ trợ 24/7.

Ví dụ thực tế: Hệ thống API Payment của bạn đang chạy trong dịp Black Friday bị một botnet tấn công bằng SYN flood với hàng triệu request mỗi giây. SGs/NACLs sẽ bị nghẽn băng thông, nhưng AWS Shield Advanced sẽ hấp thụ và phân tán lượng traffic rác này trước khi nó kịp chạm tới VPC của bạn, giữ cho API vẫn phản hồi tốt.

# 2. AWS WAF (Web Application Firewall)
Chi tiết: Tường lửa bảo vệ ở tầng Ứng dụng (Layer 7). Nhiệm vụ chính là lọc lưu lượng HTTP/HTTPS độc hại dựa trên các quy tắc bạn thiết lập (như chặn IP cụ thể, chặn request có chứa mã độc).

Ví dụ thực tế: Hacker cố gắng gửi các payload chứa mã độc (SQL Injection) vào REST API Endpoint của bạn. AWS WAF sẽ quét nội dung HTTP body và chặn đứng request này trước khi nó chạm tới server Node.js hoặc Rust của bạn.

# 3. Amazon API Gateway
Chi tiết: Cổng giao tiếp cho phép tạo, xuất bản, duy trì, giám sát và bảo mật các API (REST, HTTP, WebSocket) ở mọi quy mô. Nó hỗ trợ tính năng Throttling (điều tiết lưu lượng) để giới hạn số lượng request.

Ví dụ thực tế: Bạn có một kiến trúc Microservices giao tiếp qua gRPC hoặc REST. Để tránh việc database bị quá tải khi lượng người dùng tăng đột biến (traffic spikes), bạn cấu hình API Gateway giới hạn chỉ cho phép tối đa 10,000 requests/giây. Các request vượt mức sẽ nhận mã lỗi 429 (Too Many Requests), giúp backend server sống sót.

Nhóm Lưu Trữ (Storage)
# 4. Amazon S3 (Simple Storage Service)
Chi tiết: Dịch vụ lưu trữ đối tượng (Object Storage). Cực kỳ rẻ, mở rộng vô hạn và độ bền cao. Phù hợp lưu trữ file tĩnh, backup, tài liệu. Hỗ trợ S3 Lifecycle Rules để tự động chuyển lớp lưu trữ hoặc xóa file sau một thời gian cấu hình.

Ví dụ thực tế: Triển khai một ứng dụng Next.js, bạn cho phép người dùng upload ảnh avatar. Các ảnh này được lưu trực tiếp vào S3. Bạn cấu hình S3 Lifecycle để tự động xóa các file tạm trong thư mục /temp sau 30 ngày để tiết kiệm chi phí.

# 5. Amazon EFS (Elastic File System)
Chi tiết: Hệ thống tệp mạng được quản lý hoàn toàn cho môi trường Linux. Hỗ trợ chuẩn POSIX, cho phép hàng ngàn máy chủ truy cập đồng thời.

Ví dụ thực tế: Bạn có 10 container Docker chạy backend API. Cả 10 container này cần đọc/ghi chung vào một thư mục chứa log hệ thống hoặc các file báo cáo PDF đang được render chung. EFS được mount vào tất cả container, cho phép chúng thao tác với file như trên cùng một ổ cứng cục bộ.

# 6. Amazon EBS (Elastic Block Store)
Chi tiết: Ổ lưu trữ dạng khối, được thiết kế để gắn vào một máy ảo EC2 duy nhất tại một thời điểm (tương tự như ổ cứng SSD gắn trong laptop). Tốc độ đọc/ghi (IOPS) rất cao.

Ví dụ thực tế: Cài đặt một cơ sở dữ liệu PostgreSQL hoặc Kafka broker tự quản lý trên máy ảo EC2. Bạn sẽ gắn ổ EBS vào máy ảo này để chứa dữ liệu DB vì nó cần tốc độ truy xuất cực nhanh ở cấp độ block (khối dữ liệu).

# 7. Amazon FSx for Windows File Server
Chi tiết: Hệ thống tệp chia sẻ gốc (native) cho Windows, sử dụng giao thức SMB.

Ví dụ thực tế: Công ty có một hệ thống ERP cũ viết bằng .NET chạy trên Windows Server on-premise, yêu cầu lưu file báo cáo vào một Share Folder dạng \\server\share. Khi đưa lên AWS, dùng FSx for Windows giúp không phải viết lại code của ứng dụng.

Nhóm Máy Tính & Kiến Trúc (Compute & Architecture)
# 8. AWS Transfer Family
Chi tiết: Dịch vụ quản lý hoàn toàn cho phép truyền file qua các giao thức SFTP, FTPS, và FTP trực tiếp vào Amazon S3 hoặc EFS.

Ví dụ thực tế: Đối tác logistics yêu cầu gửi file CSV đối soát tự động hàng ngày qua giao thức SFTP. Bạn không cần dựng máy chủ Ubuntu rồi cài OpenSSH thủ công. Chỉ cần bật AWS Transfer, cấp tài khoản, file sẽ được đối tác đẩy thẳng vào S3 bucket một cách an toàn.

# 9. AWS Lambda
Chi tiết: Dịch vụ điện toán serverless (không máy chủ). Cho phép chạy code mà không cần cấp phép hay quản lý máy chủ. Bị tính tiền theo thời gian thực thi (miliseconds).

Ví dụ thực tế: Khi có một bức ảnh được upload vào S3, một hàm Lambda tự động được kích hoạt (trigger) để chạy script resize ảnh thành các kích cỡ nhỏ hơn làm thumbnail, sau đó hàm tự tắt.

# 10. Amazon Aurora
Chi tiết: Cơ sở dữ liệu quan hệ (Relational Database) do AWS thiết kế, tương thích hoàn toàn với MySQL và PostgreSQL. Tốc độ cao hơn và khả năng tự động sao chép lưu trữ ra nhiều vùng (Availability Zones) để đạt tính sẵn sàng cao (HA).

Ví dụ thực tế: Hệ thống lõi xử lý giao dịch cần một database PostgreSQL mạnh mẽ. Thay vì tự cài đặt và lo cấu hình Master-Slave phức tạp, bạn dùng Amazon Aurora PostgreSQL. Nếu Master node chết, Aurora tự động thăng cấp một Replica lên làm Master trong vòng vài giây mà không làm gián đoạn ứng dụng.
