# Fine grained IAM user permission
> ĐỊnh nghĩa: cung cấp kiểm soát truy cập chi tiết, quyền hạn tối tối thiểu, giới hạn người dùng với những hành động chỉ định trên tài nguyên chỉ định hơn là broad access (tiếp cận đa dạng)
> Đạt được bằng cách sử dụng JSON policies cái mà định nghĩa các hành động được cho phép, tài nguyên và điều kiện

# AWS Lambda
> Định nghĩa: là một serverless, dịch vụ điện toán hướng sự kiện, cho phép bạn chạy code không cần cung cấp hoặc hoặc lí máy chủ, dễ dàng upload code, và lambda handle mọi thứ được yêu cầu để chạy và scale nó với HA

# AWS Key Management
> Định nghĩa: là một dichj vụ quản lý, đươn giản dể tạo và kiểm soát khóa mật mã được sử dụng để kí dữ liệu qua AWS và app

# Amazon RDS (Relational Database Service)
> Định nghĩa: Là dịch vụ quản lý cơ sở dữ liệu quan hệ, giúp bạn dễ dàng thiết lập, vận hành và mở rộng quy mô các cơ sở dữ liệu quen thuộc như MySQL, PostgreSQL, hoặc SQL Server trên đám mây.

# Amazon CloudWatch
> Định nghĩa: Là một dịch vụ giám sát và quản lý hệ thống, thu thập các logs, metrics và events từ các tài nguyên AWS, giúp bạn phản ứng nhanh chóng với những thay đổi về hiệu suất của ứng dụng.

# Amazon CloudFront
> Định nghĩa: Là một dịch vụ mạng phân phối nội dung (CDN) toàn cầu, giúp truyền tải dữ liệu, video, ứng dụng và API đến người dùng cuối một cách an toàn với độ trễ thấp và tốc độ cao.

# Amazon Route 53
> Định nghĩa: Là dịch vụ Hệ thống phân giải tên miền (DNS) trên đám mây có tính khả dụng cao, kết hợp với tính năng kiểm tra tình trạng (health check) và định tuyến lưu lượng thông minh.

# Amazon ECS (Elastic Container Service)
> Định nghĩa: Là dịch vụ điều phối vùng chứa (container orchestration) được quản lý hoàn toàn, giúp bạn dễ dàng triển khai, quản lý và thay đổi quy mô các ứng dụng chạy bằng Docker.

# AWS Fargate
> Định nghĩa: Là công cụ điện toán serverless dành cho container (hoạt động cùng với ECS hoặc EKS), cho phép bạn chạy các container mà không cần phải quản lý các máy chủ EC2 bên dưới.

# AWS CloudFormation
> Định nghĩa: Là dịch vụ cho phép bạn sử dụng Infrastructure as Code (Cơ sở hạ tầng dưới dạng mã) để mô hình hóa, cung cấp và quản lý tất cả các tài nguyên đám mây thông qua các file template (JSON/YAML).

# Amazon SQS (Simple Queue Service)
> Định nghĩa: Là dịch vụ hàng đợi tin nhắn (message queuing) được quản lý hoàn toàn, cho phép bạn phân tách các thành phần của ứng dụng (microservices) để chúng giao tiếp với nhau một cách bất đồng bộ và an toàn.

# Amazon SNS (Simple Notification Service)
> Định nghĩa: Là dịch vụ nhắn tin theo mô hình pub/sub (publish/subscribe), dùng để gửi tin nhắn hàng loạt từ hệ thống đến hệ thống (như gửi tới SQS, Lambda) hoặc gửi thông báo trực tiếp cho người dùng (SMS, Email).

# Amazon ElastiCache
> Định nghĩa: Là dịch vụ lưu trữ dữ liệu trong bộ nhớ (in-memory data store) được quản lý hoàn toàn, tương thích với Redis và Memcached, giúp tăng tốc hiệu suất ứng dụng bằng cách giảm độ trễ truy cập database.
