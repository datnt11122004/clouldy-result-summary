câu 1: A company wants to migrate its on-premises data center to AWS. According to the company's compliance requirements, the company can use only the ap-northeast-3 Region. Company administrators are not permitted to connect VPCs to the internet.
Which solutions will meet these requirements? (Choose two.)

Tóm tắt: Đề hỏi di chuyển data center từ om-prem lên aws với ràng buộc là chỉ được sử dụng region ap-northeast-3 và quản trị viên khong được kết nối VPC với internet(tức là ko cho phép tạo hoặc là gắn  IG, NAT GW,hoặc bất kì kết nối outbound/inbound nào dẫn ra internet)

-> bài học rút ra: 
- AWS Control tower cung cấp data residency guardrails để tự động chặn deploy tài nguyên ngoài region đã chỉ định
- AWS WAF chỉ bảo vệ web/app (http/https) tại ALB/Cloudfront, ko chặn internet access cho VPC, "aws account region" ko có tính năng deny region
- SCP (service control policy) - là thằng cấp quyền ở orgnization level, là công cụ chính để deny cross-region  và block VPC internet
- AWS Config chỉ detect và alert, ko prevent (ko chặn tạo IGW or deploy ngoài region chỉ định)

caua 4: Organizers for a global event want to put daily reports online as static HTML pages. The pages are expected to generate millions of views from users around the world. The files are stored in an Amazon S3 bucket. A solutions architect has been asked to design an efficient and effective solution.


tóm tắt: có 1 tổ chức muốn đăng 1 report daily lên mạng dưới dạng html, giờ ông SA đng được hỏi về thiết kế giải pháp hiệu quả và tối ưu 

--> rút ra:
- S3 mặc định chỉ phục vụ nội dung từ 1 region, gây độ trễ cao và tải nặng nếu hàng triệu user truy cập trực tiếp từ khắp nơi. 
- amazon cloudfront là dịch vụ CDN của aws, chuyên phân phối static content từ S3 1 cách toàn cầu và scale tự động
+ nó sừ dụng hàm trăm edge locations trên thế giới để cache nội dung gần user nhất, giảm latency xuống mức thấp nhát (<100ms ), chịu được hàng triệu RPS mà ko ảnh hưởng s3 gốc
+ tích hợp hoàn hảo với s3: s3 làm origin, CloudFront tự động invalidate cache khi file thay đổi (qua s3 event hoặc lamda@edge)
+tiết kiệm chi phí: Cache hit gimar chuyển dữ liệu. ra ngoài s3, chỉ tính phí theo GB served và requests
+ 2026 nó hỗ trợ Origin Acccess control (OAC) mới nhát để bảo mật s3 private bucket
----> nhắc đến serve static page -> auto nghĩ đến cloudfront làm CDN + s3 

- Presign URL chỉ cho truy cập tạm thời (tạm thời, thường 7-15p) dến private s3 object, phù hợp chia sẻ cá nhân hóa chứ ko phải public static page. Với million views, phải generate URL mới liên tục (tốn compute), ko cache, độ trễ cao, ko scale toàn cầu ----> k hiệuq ủa cho web public 

- Cross-region replication (CRR) copy data S3 sang tất cả Regions (24+ regions), tốn kém storage/replication fees lớn (hàng TB data x millions views), S3 + cloudfront đã global mà ko cần replicate(cloudfront pull từ orgin 1 region)
- Geoproximity của route 53 (dịch vụ DNS) dựa trên vị trí địa lý (bias traffic đến nearer endpoint), ko cache content hay phân phối file, vẫn phải serve từ s3 single region -> latency cao với static pages lớn, route 53 chỉ layer7 DNS, ko thayu thế CDN như cloudfront (User → Route 53 → S3 (1 region duy nhất))

câu 5:A company is migrating its on-premises PostgreSQL database to Amazon Aurora PostgreSQL. The on-premises database must remain online and accessible during the migration. The Aurora database must remain synchronized with the on-premises database.
Which combination of actions must a solutions architect take to meet these requirements? (

tóm tắt: 1 cty đag chuyển dữ liệu ở PostgreSQL từ on-prem lên Amazon Aurora PostgreSQL, db on-prem vẫn phải luôn hoạt động và có thể truy cập trong suốt quá trình migration.
DB Aurora cũng phải dc đồng bộ liên tục với db on-prem

-> rút ra bài học:
- nếu cần  sao chép dữ liệu liên tục (ongoing replication)  + ko gián đoạn ---> auto sử dụng AWS database migration service (AWS DMS), nó hỗ trợ full load + change data capture (CDC) và apply các thay đổi liên tục (insert/update/delete) từ source (db on-prem) sang target (db trên aurora) thời gian thực 

- Để migration liên tục mà không downtime, AWS DMS yêu cầu:

+ Tạo DMS replication instance (gọi là replication server): Đây là server trung gian (EC2-based hoặc Serverless DMS mới từ 2023+) để xử lý replication giữa source (on-premises PostgreSQL) và target (Aurora PostgreSQL).
+ Tạo ongoing replication task: Task này bao gồm full load ban đầu + CDC (ongoing replication) để capture và apply các thay đổi (insert/update/delete) từ source sang target thời gian thực.
Kết hợp hai bước này đảm bảo Aurora luôn sync với on-premises. Sau khi sync ổn định, switchover bằng cách cập nhật application connection string sang Aurora (cutover).
---->  Hoàn hảo phù hợp yêu cầu "remain online and synchronized".
